浏览量重复统计bug修复

2021-04-19 2 min read # Gridea

我就说我的博客不会有那么多人看嘛😩

👀 views

问题

每点一次任何一篇文章都会给所有文章加访问量。

原因

依然在调查。

解决

网站总体

仅在主题的footer.ejs或header.ejs中加入

<script async src="//busuanzi.ibruce.info/busuanzi/2.3/busuanzi.pure.mini.js"></script>

使用Gridea写博客的话该文件保存在

/Users/[用户名]/Documents/Gridea/themes/[主题名]/templates/includes

其中/Users/[用户名]/Documents/Gridea为其Mac客户端的默认目录。

每一页

显示站点总访问量

仅在关于页面中加入
算法a：pv的方式，单个用户连续点击n篇文章，记录n次访问量。

<span id="busuanzi_container_site_pv">本站总访问量<span id="busuanzi_value_site_pv"></span>次</span>

算法b：uv的方式，单个用户连续点击n篇文章，只记录1次访客数。

<span id="busuanzi_container_site_uv">本站总访客数<span id="busuanzi_value_site_uv"></span>人</span>

显示单页面访问量

在每一篇文章的页面加入
算法：pv的方式，单个用户点击1篇文章，本篇文章记录1次阅读量。

<span id="busuanzi_container_page_pv">本文总阅读量<span id="busuanzi_value_page_pv"></span>次</span>

汉字均可替换成其他。

结果

修了个🔨
原来的错误结果也没清零，bug也还在那。
这个问题暂时搁置，改日再说。

改日

放弃卜算子了，换statcounter，访客IP统计结果会定期发到我的邮箱。
试试这个准不准（不过你们也看不到，只有我能看见结果）。

网站bug修复贴统一发布在自带的Gridea标签下。
