博客GitHub联网问题修复

发布于 2021-04-15 | 标签: 公告 | 2分钟 | 484字数

对大家浏览博客无影响，只是让我的更新上传工作更快了。💃
博文封面图也是用来测试代理的，如果正常显示了图片证明了你打开了代理（或者你不在中国大陆）。🕺

常出现问题

Gridea在检测远程连接时经常报

ERROR: connect ETIMEOUT
一个GitHub的IP

导致我经常无法及时更新博客。

产生原因

  1. Github在国内的联网很慢
  2. 软件内使用的.git配置的链接无法从github改为镜像站
  3. 其他VPN或代理软件无法对其进行代理操作
  4. 作者只给出了“🙁 同步遇到了错误，请查阅 FAQ 或 Issues 来寻找解决方案”，然而官网FAQ中并没有解决办法
  5. 尝试了Issues中修改Hosts增加可ping通的Github IP的操作，但效果甚微。

解决办法

通过Proxifier软件单独对某一个app，如Gridea进行代理。（前提是已有可用代理）
我用的是Mac，PC上的操作相似，下面的软件也都是双平台均可用的。

  1. Proxifier可用Homebrew或在官网下载
    brew install proxifier
    三代Mac版
    用户名
    macwk.com
    激活码
    2DNRX-V3PNK-TEGYN-DR01D-9UGGT
  2. 打开Proxifier，点击Proxies添加代理，写入代理与端口号。

    从上至下依次为：蓝灯、ShadowsocksX-NG-R8、ClashX。
    查看常用的三个fanqiang软件代理地址方法：
    • 蓝灯
      账号-设置-显示高级设置
    • ShadowsocksX-NG-R8
      高级设置
    • ClashX
      Dashboard-设置
  3. 进入旁边的Rules为Gridea设置代理

    相似地，也可以为Twitter与Telegram等软件单独设置代理。
  4. 同步博文的时候注意要保持代理软件开启
    三代可以设置关闭后继续使用代理配置。
